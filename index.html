WHAT IS EC2?

Amazon Elastic Compute Cloud, EC2 is an Amazon web service that provides sizable compute services in the cloud. They are resizable and can quickly scale or reduce the number of instances the server is using if computing requirements change.

ADVANTAGES

  • Rapid Deployment

    Deploy your application in minutes with a few clicks. The usage of cloud computing reduces several time-consuming activities.

  • Cost reduction

    As capital costs are transformed into operational maintenance costs, costs are dramatically reduced

  • Reliability

    Cloud users are contracted to use with very high quality of service terms written in the contract

  • Flexibility

    Elasticity represents flexibility in providing computing resources according to the actual needs of users or service applications

  • Sharing

    Renting and sharing resources between reducing the cost of infrastructure investment among a large range of users

  • Multimedia Access

    Access using web browser regardless of location or the device, such as a PC or mobile

  • Enhanced Load Capacity

    Adding computational power to high loads can be done with mouse clicks or be automated

  • Security

    As the data concentration in the clouds grows larger, providers are always focus on improving technology and setting barriers to increase data security

DISADVANTAGES

Cost

Reducing initial investment costs is the advantage of cloud computing. However, it is also a controversial issue as cloud users always have to pay for the service. Compared to investing in infrastructure on their own, cloud users have no assets after amortization of investment costs.

Monitoring and management tools

The monitoring and maintenance tools are incomplete and the ability to communicate with the cloud is limited, although BMC, CA, Novell have announced that these applications data center management is being enhanced to provide greater control over data in Amazon EC2 cloud and cloud services.

Cloud standardization

Standardization of cloud design and communication has not been adopted. Each platform provides different API application communication and management interfaces. Today, organizations such as the Distributed Management Task Force, the Cloud Security Alliance, and the Open Cloud Foundation are developing standards for interoperability management, data migration, security, and other cloud capabilities.

Availability

Availability is the advantage of the cloud in theory. However, in reality with current clouds, availability is sometimes not guaranteed and is also an obstacle nowadays, when only a few service providers are committed to availability and continuity of service, in terms of repair time and data recovery.

Contract compliance has also become complicated

Cloud service providers can move data to other countries with cheaper electricity, but looser laws that allow users of computing services to not informed. This is possible because the cloud is transparent to the user.

Privacy

Most contracts represent a contract between a cloud provider and user that promises a scenario in which customer data is always safe and private. However, privacy in cloud computing is also an issue of concern because information security infrastructure for the cloud is still a topic of research in the scientific world.

Service level

Cloud computing provides services on demand, but in reality, service packages are often predefined and users base on needs and capabilities to choose available services. . For example, the detailed configuration of virtual machine parameters is not currently possible.

The ability to integrate with the organization's existing information infrastructure

The integration of cloud computing into the existing infrastructure of the customer does not have a specific model and implementation method. Private cloud and commercial cloud connectivity models are still being studied.
